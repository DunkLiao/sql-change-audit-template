# AI 改 SQL Prompt 模板

## 基本版（每次必貼）
請依下列規則修改 SQL：
1. 修改後，請列出「預期會影響的欄位」與「預期會影響的資料範圍」
2. 不影響的欄位請保證輸出與舊版完全一致
3. 若有調整常數，請明確列出新舊對照
4. 若有調整 CASE WHEN 分支，請保留 CASE_BRANCH 標記欄位
5. 改完後請說明「我預期 Diff Report 會看到的差異有哪些」
6. 不確定的需求請先提問再動手
7. 一次只改一件事

## 嚴格版（涉及金額/法遵時）
- 列出本次改動的業務動機與預期效果
- 列出將修改的檔案/View/函數清單
- 列出將影響的欄位與資料範圍（用 WHERE 條件描述）
- 列出不應該被影響的欄位與範圍
- 不允許硬編碼測試資料
- 涉及 NULL 處理時，必須明確說明 NULL 該被視為哪一類
- 禁止一次改動超過 1 個邏輯主題
- 禁止刪除原有註解
- 禁止重新命名既有欄位

## 自審版（改完後讓 AI 自我審查）
請以「程式碼審查者」角色重新檢視你剛才的修改：
1. 有沒有任何欄位被你「順手」改了，但並非本次需求範圍？
2. 有沒有任何 CASE WHEN 分支判斷順序被改動？
3. 有沒有任何 JOIN/GROUP BY/WHERE 被改動？
4. 有沒有任何 NULL 處理被改動？
請逐項回答並對「有」的部分說明原因。
